# 迁移运行前检查说明

这份说明只用于帮助你在正式迁移前做检查。

现在不要写假密钥，不要伪造真实迁移数据。

## 1. `migrate/import.js` 运行前必须准备哪些文件

必须准备：

- [package.json](E:/rmm-2sys/2.V2rmm/package.json)
- [migrate/serviceAccount.json](E:/rmm-2sys/2.V2rmm/migrate/serviceAccount.json)
- [migrate/export.json](E:/rmm-2sys/2.V2rmm/migrate/export.json)

其他可能依赖：

- 已安装的 `node_modules`
- 可用的 Node.js 运行环境
- `FIREBASE_STORAGE_BUCKET` 环境变量
- Google Drive 读取权限
- Firebase 项目的 Firestore / Storage 可写权限

## 2. 每个文件分别是干什么的

`package.json`

- 告诉 Node 这个项目使用 ES Module
- 声明迁移脚本依赖哪些包
- 让你可以先安装依赖再运行脚本

`migrate/serviceAccount.json`

- 这是 Firebase 服务账号密钥文件
- 脚本用它连接 Firebase Admin SDK
- 也可能用它去读取 Google Drive 文件

`migrate/export.json`

- 这是旧系统导出的真实迁移数据
- 脚本会读取它，把数据写入 Firestore
- 里面还会包含凭证图片对应的旧 ID 信息

`node_modules`

- 里面是安装好的依赖包
- 没有它，脚本导入 `firebase-admin` 和 `googleapis` 会失败

## 3. 哪些文件必须是真实文件，不能伪造

下面这些必须是真实文件：

- [migrate/serviceAccount.json](E:/rmm-2sys/2.V2rmm/migrate/serviceAccount.json)
- [migrate/export.json](E:/rmm-2sys/2.V2rmm/migrate/export.json)

原因：

- `serviceAccount.json` 如果是假的，脚本无法真正连接 Firebase
- `export.json` 如果是假的，迁移出来的数据就是错的，风险很高

下面这些可以先只有模板或骨架：

- 但真正执行迁移前，依赖必须已经安装成功

## 5. 如果正式开始迁移，最安全的执行顺序

建议顺序：

1. 先确认 [package.json](E:/rmm-2sys/2.V2rmm/package.json) 已存在，Node.js 可用
2. 安装依赖，让 `firebase-admin` 和 `googleapis` 可用
3. 准备真实的 [migrate/serviceAccount.json](E:/rmm-2sys/2.V2rmm/migrate/serviceAccount.json)
4. 从旧系统导出真实的 [migrate/export.json](E:/rmm-2sys/2.V2rmm/migrate/export.json)
5. 检查 `export.json` 不是空文件，结构正常
6. 设置真实的 `FIREBASE_STORAGE_BUCKET`
7. 先执行一次 `DRY_RUN=1` 的测试迁移
8. 先看输出日志，确认没有大错误
9. 再执行正式迁移
10. 最后检查迁移报告和 Firebase 中的数据

## 6. 哪些步骤最容易出错，会报什么类型的错

依赖没装好

- 常见报错类型：
  `Cannot find package`
  `Cannot find module`
  `ERR_MODULE_NOT_FOUND`

`serviceAccount.json` 不存在

- 常见报错类型：
  找不到文件
  读取文件失败
  权限不足

`serviceAccount.json` 内容不对

- 常见报错类型：
  JSON 解析失败
  Firebase 凭证初始化失败
  权限认证失败

`export.json` 不存在或为空

- 常见报错类型：
  找不到文件
  JSON 解析失败
  读取到空内容

`FIREBASE_STORAGE_BUCKET` 没设置

- 常见报错类型：
  脚本前置检查直接失败

Google Drive 权限不够

- 常见报错类型：
  403
  404
  无法下载图片

Firebase 权限不够

- 常见报错类型：
  写入 Firestore 失败
  上传 Storage 失败
  权限拒绝

网络不稳定

- 常见报错类型：
  超时
  请求失败
  部分图片迁移失败

## 最后提醒

正式迁移前，最重要的是三件事：

- `serviceAccount.json` 必须是真实可用的
- `export.json` 必须是真实完整的
- 一定先跑一次测试模式，再跑正式迁移
