# 环境配置说明

这份文件只用于说明项目以后怎么填配置。

现在不要写真实密钥，不要把敏感信息放进前端。

## 前端需要哪些配置字段

前端当前使用这些字段：

- `WORKER_URL`
  前端请求 Cloudflare Worker 的地址
- `FIREBASE_API_KEY`
  Firebase Web App 配置字段
- `FIREBASE_AUTH_DOMAIN`
  Firebase Web App 配置字段
- `FIREBASE_PROJECT_ID`
  Firebase 项目 ID
- `FIREBASE_STORAGE_BUCKET`
  Firebase Storage bucket
- `FIREBASE_MESSAGING_SENDER_ID`
  Firebase Web App 配置字段
- `FIREBASE_APP_ID`
  Firebase Web App 配置字段
- `GAS_V1_URL`
  老系统 GAS 地址，过渡期才需要
- `SHADOW_WRITE_ENABLED`
  是否开启双写日志
- `DUAL_BLIND_BANNER`
  是否显示双盲核对横幅

## Worker 需要哪些环境变量

Worker 代码当前需要这些变量：

- `GEMINI_API_KEY`
- `FIREBASE_PROJECT_ID`
- `FIREBASE_FUNCTIONS_URL`
- `WORKER_INTERNAL_SECRET`

## 哪些字段以后去 Firebase 控制台拿

这些字段通常去 Firebase 控制台获取：

- `FIREBASE_API_KEY`
- `FIREBASE_AUTH_DOMAIN`
- `FIREBASE_PROJECT_ID`
- `FIREBASE_STORAGE_BUCKET`
- `FIREBASE_MESSAGING_SENDER_ID`
- `FIREBASE_APP_ID`

常见位置：

- Firebase 控制台
- Project settings
- Your apps
- Web app 配置

## 哪些字段以后去 Cloudflare Worker Secrets 设置

这些应放到 Cloudflare Worker Secrets：

- `GEMINI_API_KEY`
- `WORKER_INTERNAL_SECRET`

这些通常也应在 Worker 环境里配置：

- `FIREBASE_FUNCTIONS_URL`
- `FIREBASE_PROJECT_ID`

## 哪些字段绝对不能写进前端

绝对不要写进前端页面或前端 JS：

- `GEMINI_API_KEY`
- `WORKER_INTERNAL_SECRET`
- `serviceAccount.json` 里的任何内容
- 任何管理员私钥
- 任何后端服务密钥

## 哪些字段可以出现在前端

这些 Firebase Web App 字段通常可以放前端，但必须填真实项目对应值：

- `FIREBASE_API_KEY`
- `FIREBASE_AUTH_DOMAIN`
- `FIREBASE_PROJECT_ID`
- `FIREBASE_STORAGE_BUCKET`
- `FIREBASE_MESSAGING_SENDER_ID`
- `FIREBASE_APP_ID`
