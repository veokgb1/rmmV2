## F. 交易与凭证关联核对

### 正向核对：transaction -> voucher

- [ ] 每条有 `voucherIds` 的交易都能找到对应 voucher 文档
- [ ] 每条交易的 `voucherRefIds` 只包含已成功落库的 voucher
- [ ] 每条交易的 `voucherStoragePaths` 只包含已成功上传的图片路径
- [ ] `voucherRefIds` 无重复
- [ ] `voucherStoragePaths` 无重复

### 反向核对：voucher -> transaction

- [ ] 每个 voucher 的 `linkedTransactionIds` 都能找到对应 transaction 文档
- [ ] 每个 voucher 的 `linkedTransactionIds` 与 `linkedTransactionKeys` 对应一致
- [ ] 已成功迁移的 voucher 至少关联到一条合法 transaction，或已明确标记为特殊情况

### 双向一致性核对

- [ ] transaction 引用的 voucher，voucher 端也能反查回该 transaction
- [ ] voucher 反查到的 transaction，transaction 端也包含该 voucher
- [ ] 没有单向成功、另一侧缺失的情况
